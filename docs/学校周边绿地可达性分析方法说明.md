# 学校周边公园绿地可达性分析——方法说明

## 一、分析目标

量化海淀区156所小学步行500米范围内的公园绿地可达性，获取以下信息：可到达的公园数量与名称、步行至最近公园的实际距离与时间。

## 二、数据来源

### 2.1 学校坐标数据

以本研究156所小学的AOI面状边界数据为基础，计算各校多边形质心坐标（WGS-84）。经GCJ-02坐标偏移算法转换后，用于调用高德地图Web服务API。

坐标转换采用国家测绘局公布的GCJ-02加密算法，该算法通过非线性偏移函数对WGS-84经纬度进行扰动，北京地区偏移量约为300至500米。

### 2.2 公园绿地POI数据

调用高德地图"周边搜索"接口（`/v3/place/around`），以每所学校质心为中心，3000米为搜索半径，查询类型编码为`110101`（公园）和`110103`（植物园）的兴趣点。接口返回公园的名称、类型、坐标及与搜索中心的直线距离。

选择3000米搜索半径而非500米，是为了确保即使学校周边500米内没有公园，也能从更大范围内找到最近的公园并计算步行距离。实际分析中，500米范围内的公园通过本地筛选单独统计。

高德地图POI分类体系中，`110101`涵盖各类综合性公园、社区公园和街头绿地，`110103`对应植物园。两类是城市中最典型的、可供市民日常使用的公共绿地空间。

### 2.3 步行路径数据

对搜索到的公园，按直线距离由近到远排序，取每所学校最近的5个公园，调用高德地图"步行路径规划"接口（`/v3/direction/walking`），获取实际步行距离（米）与步行时间（秒）。该接口基于高德道路网络数据计算，反映沿城市人行道路的实际步行路径，而非直线距离。

每校仅对最近5个公园计算步行距离，是兼顾精度与效率的折中——步行距离的计算仅需确认最近公园即可满足分析需求，无需对所有候选公园逐一计算。

## 三、指标定义

| 指标 | 定义 | 数据来源 |
|------|------|----------|
| 500m内公园数量 | 以学校为中心、直线距离500m为半径范围内的公园POI个数 | 高德周边搜索（本地按直线距离筛选） |
| 500m内公园名称 | 上述范围内公园的具体名称列表 | 同上 |
| 最近公园名称 | 步行距离最近的公园名称 | 高德步行路径规划 |
| 最近公园步行距离 | 学校至最近公园的实际步行路程（米） | 高德步行路径规划 |
| 最近公园步行时间 | 学校至最近公园的步行预计用时（秒） | 高德步行路径规划 |

500米对应约6至8分钟步行路程，参照《城市居住区规划设计标准》（GB 50180-2018）中"五分钟生活圈"的服务半径设定。

## 四、技术路线

```
156所小学AOI面状边界
        │
        ▼
计算多边形质心（WGS-84）
        │
        ▼
坐标转换（WGS-84 → GCJ-02）
        │
        ▼
逐校调用高德周边搜索API（半径3000m，类型110101|110103）
        │
        ├─→ 本地筛选直线距离≤500m的公园 → 统计数量与名称
        │
        ▼
按直线距离排序，取最近5个公园
        │
        ▼
逐公园调用高德步行路径规划API
        │
        ▼
获取实际步行距离与时间
        │
        ▼
取步行距离最短的公园 → 输出最近公园信息
```

## 五、方法局限性

1. **POI点位代表性**：高德地图中公园POI坐标通常为标注点（可能为入口、管理中心或几何中心），而非公园边界。步行距离计算结果可能存在偏差——实际到达公园入口的步行距离可能更短或更长。

2. **直线距离筛选的近似性**：500米服务范围以直线距离（欧氏距离）划定，而非步行距离。同一500米直线半径内，沿主干道方向的实际步行范围可能大于沿无路方向。

3. **POI覆盖完整性**：高德地图POI数据受采集更新周期影响，部分小型社区绿地或新建公园可能存在遗漏。

4. **步行时间估算**：步行时间基于高德地图默认步行速度（约4.8km/h）计算，未考虑使用者年龄差异。小学生实际步行速度可能低于成人平均值。

5. **计算范围限制**：每校仅对最近5个公园计算步行距离，若最近公园恰好排在第6位及以后（极罕见情况），可能未被纳入步行距离计算。
